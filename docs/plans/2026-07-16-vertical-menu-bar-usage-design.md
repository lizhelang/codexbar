# 菜单栏百分比纵向内嵌设计

## 目标

在保持 `NSStatusItem.squareLength` 的前提下，把可选百分比移入 18×18 pt 图标顶部，进度条放在图标下方。无论是否开启百分比，菜单栏都只占用一个方形状态项，不再因文字横排而变宽。

## 已确认的展示规则

- 顶部只显示最短的实际额度窗口百分比。
- 若上游返回 5h 与 7d，顶部显示 5h；底部依次显示 5h、7d 两条进度。
- 若上游只返回 7d，顶部显示 7d；底部只显示一条 7d 进度，不绘制虚假的 5h 占位。
- 窗口及其顺序继续完全来自 `TokenAccount.usageWindowDisplays(mode:)`，不根据套餐类型写死 5h 永久存在或永久取消。
- 百分比与进度填充都跟随 `usageDisplayMode`，统一表达“已用”或“剩余”。

## 状态项布局

- 开启“在图标内显示当前百分比”时，图标内部使用“顶部数值、底部进度”的纵向布局。
- 关闭该设置时，沿用现有纯进度图标布局，让进度条使用更大的可绘制区域。
- 两种模式都固定使用 `squareLength` 与 `.imageOnly`，按钮标题保持为空。
- 保留现有持久化键 `showsMenuBarUsageText`，避免旧配置迁移；只调整其界面文案与渲染语义。

## 状态优先级与兼容性

- 更新、封禁、额度耗尽、视觉预警和兼容供应商继续优先显示现有系统符号。
- 系统符号状态不额外绘制百分比，也不会因设置开启而扩宽。
- 聚合网关模式仍使用实际路由账号的数据；普通模式仍使用当前激活账号。
- 无额度数据时退回系统符号，不将未知数据画成 0%。

## 可访问性

- VoiceOver 值继续包含所有实际窗口及“已用/剩余”语义，例如“5h 已用 67% · 7d 已用 48%”。
- 顶部只显示一个百分比是视觉压缩策略，不减少辅助功能可获得的信息。

## 测试与验收

- 锁定纯进度模式现有像素坐标，防止本次变化破坏默认图标。
- 新增百分比内嵌模式的单窗口、双窗口布局及“只取首个实际窗口”测试。
- 展示模型测试确认设置开启后仍为方形、标题为空，且系统符号状态不扩宽。
- 完成定向测试与全量测试后，构建并替换 `/Applications/codexbar.app`。
- 在真实菜单栏开启设置，确认状态项仍为 24×24，并核对当前上游只返回 7d 时的顶部数值和单条进度。
- 清理构建及安装残留，确保 Launch Services 只登记目标安装副本。

## 非目标

- 不改变额度计算、窗口排序、告警阈值或账号路由。
- 不在狭小图标内同时写出 `5h`、`7d` 标签。
- 不恢复横向百分比标题或可变宽度状态项。
