# Codexbar 新图标探索设计

Date: 2026-04-15

## 背景

当前 `codexbar` 的 AppIcon 能表达“代码 + 用量”语义，但整体观感偏轻、偏散，缩小后识别点不够集中，也不够像一个成熟的 macOS 菜单栏工具。

这次探索不直接替换正式图标，先产出一组风格统一、图形不同的候选稿，供人工挑选。

## 已确认约束

- 整体气质偏 macOS 系统工具，不做互联网产品式高饱和品牌感。
- 主色使用石墨灰 / 银灰，不延续当前蓝色主色。
- 不保留明显的代码符号，不再使用括号、终端、光标等直接暗示。
- 图形需要在小尺寸下仍有清晰主轮廓，避免依赖细节说明含义。
- 仍然保留“切换 / 激活 / 枢纽 / 选择器”这类与产品能力贴近的隐含语义。

## 设计策略

先固定统一的图标外壳，再只替换中心图形：

- 使用带轻微金属质感的圆角方形底板，保持 macOS 工具图标的稳定气质。
- 控制对比度在灰阶内部解决，不额外引入彩色高亮。
- 中心图形尽量使用 1 到 3 个强轮廓元素，优先保证 16px 到 32px 下的可读性。

这样做的好处是，评估时可以把注意力放在“符号是否合适”，而不是被不同底色或风格干扰。

## 候选方向

### A. Orbital Switch

- 中心为一个稳定圆核，外围使用两段错位环轨。
- 语义接近“系统级切换枢纽”。
- 图形集中，最容易做出高级、克制的工具气质。

### B. Twin Tiles

- 使用两块略有层级关系的面板，表达配置之间的前后切换。
- 语义最贴近“多账号 / 多 provider 切换”。
- 风险是如果处理不够收敛，会偏向通用效率工具。

### C. Dial Selector

- 以拨盘 / 选择器为核心隐喻，强调“当前激活项”。
- 最接近系统控制器的气质，形体也比较稳。
- 风险是过于通用，独特性略弱。

### D. Segmented Switch

- 把切换语义进一步收敛到“分段选择控件”。
- 系统感强，也更直接表达“当前选择”。
- 风险是 UI 控件感更强，需要靠比例和阴影避免过像设置项图标。

## 评估标准

- 小尺寸下是否还能一眼认出主轮廓。
- 是否有足够的 macOS 工具气质，而不是泛应用图标感。
- 是否还能隐约对应 `codexbar` 的“切换器”定位。
- 是否比当前图标更成熟、更耐看。

## 当前推荐

首推 `A. Orbital Switch`，因为它在“系统工具感”和“产品独特性”之间的平衡最好。

次推 `C. Dial Selector`，如果后续希望更克制、更接近原生工具图标，它会是最稳的备选。
