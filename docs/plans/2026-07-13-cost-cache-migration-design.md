# 计费缓存迁移期间保留历史显示

## 背景

统计口径升级会使旧的成本摘要、session cache 和 usage ledger 失效。当前实现立即丢弃旧摘要并返回空值，同时后台重扫全部会话日志；当日志包含超长 JSONL 行时，读取器会重复扫描已检查过的缓冲区，导致界面长时间显示 0 且后台持续高 CPU。

原始会话日志和旧摘要缓存仍然存在，本次修复不得删除或覆盖这些历史数据。

## 已批准方案

1. 对结构仍可解码、且包含历史数据的旧版成本摘要进行临时兼容：保留金额、tokens 和每日明细，但清空 `updatedAt`，使界面立即显示旧值并继续触发后台刷新。
2. 重写 JSONL 分块读取的扫描游标，使每个字节只被查找一次；处理完的前缀再统一移除，避免超长单行造成平方级重复扫描。
3. 新口径账本重建完成后，使用当前 schema 的新摘要自动替换临时旧值并写回缓存。
4. 迁移失败或尚未完成时不得把有效的旧摘要覆盖成全 0。

## 验证

- 回归测试覆盖：旧 schema 非空摘要仍可显示且会触发刷新；超长单行 JSONL 能在线性时间内完成解析；重建结果仍使用当前 schema。
- 运行全量 macOS 测试。
- 在本机 2,110 个、约 8.4 GB 的真实会话日志上重装并验证：启动可响应、旧历史立即可见、缓存迁移最终完成、CPU 恢复空闲。
- 清理临时 App 与 Launch Services 注册，只保留 `/Applications/codexbar.app`。
