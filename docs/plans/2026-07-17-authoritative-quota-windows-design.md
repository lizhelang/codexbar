# 真实额度窗口设计

## 目标

Codexbar 只展示 OpenAI usage 接口真实返回的额度窗口。账号套餐类型不得作为创建 7 天窗口的证据；刷新得到单个月度窗口时，必须清除本地旧的 7 天窗口。

## 选择的方案

以 `WhamUsageResult` 返回的窗口为唯一事实来源：

- `TokenAccount` 可以对真实窗口排序、去重、格式化，但不得根据 `planType` 新建窗口。
- 只有 `secondaryLimitWindowSeconds`、`secondaryResetAt` 或非零 `secondaryUsedPercent` 提供真实证据时，才保留兼容旧数据所需的 secondary 窗口。
- `WhamService.refreshOne` 已把最新响应的可选字段直接赋给账号；规范化过程不得重新制造被刷新清空的字段。
- 上游明确返回使用率为 0 的 secondary 窗口时，因其带有窗口周期或重置时间，仍正常展示。

## 拒绝的方案

- 按 Plus/Pro/Team 默认补 7 天窗口：会制造不存在的额度并影响排序、告警与路由。
- 仅在 UI 隐藏 0% 窗口：持久化和路由仍会使用错误数据。
- 延迟清除旧窗口：无法保证刷新结果与上游当前事实一致。

## 验收标准

1. Team/Pro/Plus 的月度单窗口账号只生成一个展示窗口。
2. 刷新前存在旧 7 天窗口、刷新后上游只返回月度窗口时，持久化结果清除旧 7 天窗口。
3. 上游真实返回短周期和长周期两个窗口时，两者均保留。
4. 现有 UI、使用量百分比模式和账号排序接口保持不变。
