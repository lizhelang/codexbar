# 紧凑菜单栏双进度图标设计

## 目标

将 Codexbar 的菜单栏默认显示从“系统图标 + 百分比文字”改为紧凑的 18×18 模板图标。图标内部使用两条横向进度条表达短周期窗口（通常为 5h）和周窗口（通常为 7d），把状态栏占宽降到单个系统图标级别。现有百分比文字保留为可选设置，用户可切回当前行为。

## 已比较方案

1. **双进度动态图标 + 可选文字（采用）**
   - 默认只显示紧凑图标；图标内仍保留两个额度窗口的信息。
   - 设置中允许恢复当前 `67%·48%` 文本。
   - 信息密度、占宽和迁移成本最平衡。
2. **静态图标**
   - 实现最简单、占宽最小，但无法在不开菜单时观察额度趋势。
3. **只保留一个百分比**
   - 比当前略窄，但仍占用文字宽度，并且无法明确区分短周期和周额度。

## 数据与展示规则

- 继续以 `TokenAccount.usageWindowDisplays(mode:)` 为额度窗口唯一来源，不在图标层硬编码 Plus/Pro 永远存在 5h 或 7d。
- 双窗口时，按现有窗口顺序绘制：上条为较短窗口，下条为较长窗口。
- 只有一个窗口时，将单条进度居中绘制；因此上游临时取消或重新加入 5h 时无需修改图标代码。
- 填充语义跟随现有 `usageDisplayMode`：设置为“已用”则填充已用比例，设置为“剩余”则填充剩余比例。
- 聚合网关模式使用当前实际路由账号的窗口；普通切换模式使用激活账号。
- 无可用额度窗口时退回现有 provider 系统图标，不虚构进度。

## 状态优先级

进度图标只表示正常额度状态。以下现有状态继续优先显示明确的系统符号：

1. 有可用更新；
2. 账号封禁；
3. 周额度耗尽；
4. 任一额度耗尽或进入视觉预警阈值；
5. 无额度数据或兼容供应商。

这样不会为了紧凑而削弱异常可见性。

## 配置与兼容性

- 在 `CodexBarOpenAISettings` 新增 `showsMenuBarUsageText: Bool`。
- 默认值与旧配置缺键解码值均为 `false`，因此升级后默认进入紧凑图标模式。
- 设置页新增“菜单栏显示百分比”开关；开启后恢复当前图标加文字展示。
- 配置通过现有 `OpenAIUsageSettingsUpdate` 保存，不触碰 `~/.codex/auth.json`，也不触发 Codex 模型配置同步。

## 绘制与布局

- 新增独立的模板图标渲染器，输出 18×18 pt、2× Retina 位图表示。
- 每条进度包含浅色轨道、描边和按比例裁切的填充；所有坐标对齐 0.5 pt 像素网格，保持菜单栏清晰。
- 紧凑模式将 `NSStatusItem.length` 设为 `squareLength`，并使用 `.imageOnly`；文字模式恢复 `variableLength` 与 `.imageLeading`。
- 图像保持 `isTemplate = true`，继续适配深浅色菜单栏和高亮状态。

## 可访问性

- 视觉上隐藏文字时，辅助功能标签仍包含当前窗口摘要，例如“Codexbar，5h 已用 67%，7d 已用 48%”。
- 无数据和异常状态继续使用现有 Codexbar 标签与明确图标。

## 测试

- 配置：默认值、旧 JSON 缺键、显式开启后的 round-trip、未知值兼容。
- 设置：draft、dirty/reconcile/save，仅修改菜单栏选项时不影响其他 OpenAI 配置。
- 展示：紧凑模式无视觉文字；文字模式保留现有摘要；聚合路由账号正确选择。
- 绘制：18×18、模板属性、单窗口/双窗口/零值/满值均可生成；异常状态不被进度图标覆盖。
- 布局：紧凑模式使用 `squareLength`，文字模式使用 `variableLength`。
- 验证：相关定向测试、完整 `xcodebuild test`，再运行安装副本做菜单栏视觉检查与残留清理。

## 非目标

- 不复制上游的角色动画、眨眼、provider 多图标系统或复杂图标主题。
- 不改变弹窗内容、额度计算、账号路由和 Wham 数据解析。
- 不以硬编码方式假设 5h 窗口永久存在或永久取消。
