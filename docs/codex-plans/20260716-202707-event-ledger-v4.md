# 逐事件计费账本 v4 实施计划

## 目标

在不改变现有 UI、Today/30 Days/Lifetime 口径和持久化历史能力的前提下，升级本地 Token 与成本统计算法，并完成提交、推送和本地重装。

## 实施步骤

1. 将 ledger 升级为逐事件保存模型、turn、tier 和来源的 v4 结构，并保留 v3 兼容解码。
2. 将 session 扫描改为逐 turn 模型归因，解析事件模型、turn ID、tier、fork 元数据和 subagent 来源。
3. 用累计高水位、`last_token_usage` 和父 fork 基线生成保守且幂等的 Token 增量。
4. 将金额计算改为逐事件重算，补齐精确模型映射、Spark 零价、Priority 价格和未知模型零价规则。
5. 自动重建仍有原始 JSONL 的 session；原始日志已消失时保留 legacy Lifetime 事件。
6. 补齐多模型、tier、fork、subagent、交错累计量、迁移、去重和历史保留测试。
7. 运行针对性测试与完整测试套件，修复所有新增回归。
8. 执行独立代码审查并处理 Critical/Important 反馈。
9. 按 Lore 协议提交并推送当前分支。
10. 构建、签名、替换 `/Applications/codexbar.app`，验证运行版本和唯一可见副本，清理构建与 Launch Services/Spotlight 残留。

## 完成标准

- 新 ledger 和扫描逻辑通过所有目标回归测试。
- 完整测试套件通过。
- 真实本机日志扫描不再把多模型 session 全部归到第一个模型。
- 提交与推送成功。
- `/Applications/codexbar.app` 为新构建且可启动。
- Spotlight、Launch Services 和磁盘检查只保留目标安装副本。
