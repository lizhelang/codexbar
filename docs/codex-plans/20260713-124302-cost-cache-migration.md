# 计费缓存迁移修复实施计划

来源请求：按方案一修复计费和用量归零，迁移期间保留历史显示。

1. 添加旧 schema 非空摘要的回归测试，证明当前实现会错误返回空摘要。
2. 添加超长单行 JSONL 的性能回归测试，证明当前读取器存在重复扫描。
3. 让旧摘要以“保留数值、清空更新时间”的形式作为迁移期间快照，并确保后台刷新仍会启动。
4. 为 JSONL 分块读取加入单调扫描游标，避免重复扫描未换行的超长缓冲区。
5. 运行针对性测试与全量测试；必要时只修复本计划覆盖的失败。
6. 构建、签名并替换 `/Applications/codexbar.app`，使用真实 8.4 GB 会话库验证显示、迁移完成时间、CPU 和主线程响应。
7. 提交并推送代码，清理临时构建 App 和 Launch Services 残留。
